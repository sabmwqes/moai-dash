# 敵のクラスデザイン

- 敵のクラスは、ゲーム内でプレイヤーが対峙する様々な敵キャラクターを定義するための設計です。各クラスは、敵の特性や行動パターンを表現し、プレイヤーに多様な挑戦を提供します。
- 戦闘時の敵の状態とは別であり，敵の基本的な特性や行動パターンを定義するためのクラスです．


## EnemyDataクラス
- 見た目：jsx elementなど？
  - 敵のビジュアルを定義する．
  - 画像は使わず，絵文字とCSSで表現する．
- 名前：string
  - red moai, blue moai, etc.
- 体力：int
- 行動：EnemyAction 二次元配列（わかりやすいようクラスに切り出してもいい）
  - 各行動は、目に仕掛けてくるスキルや攻撃を表す。
  - 1階層目を反復する．（ターンごとの行動パターン）
  - 2階層目は、どのような攻撃やスキルを仕掛けてくるかを0個以上定義する．
    - 2~8のどれに仕掛けてくるかは暫定ランダム．
    - 基本的に被らないようにする．
- EXP：int
  - 倒したときに得られる経験値．
- GOLD：int
  - 倒したときに得られるお金．
- レア度：enum
  - ノーマル、レア、エピックなどのレア度を定義する．

## EnemyActionクラス
- スキルや攻撃の種類：型未定
  - 単純なダメージ
  - 状態異常の付与
  - ダメージ＋状態異常
  - プレイヤー攻撃の軽減もしくは無効
- 対象となる目の重み：{int: weight}の辞書
  - 2~8の目に対して、どの程度の確率で攻撃やスキルを仕掛けてくるかを定義する．
  - 敵が攻撃やスキルを仕掛けてこないことも指定できる．
  - デフォルトは一様分布
